Exercise

In this session, you can either create a design for a sonification or use tools to sonify your own data.

You have options to think about and explore. You are welcome to start in advance as we only have a limited time in the session itself.

We encourage you to explore the area using the Open Sonification principles.

  1. Design a Sonification

You might want to use the time to design a sonification to begin thinking about some of the issues that the session will introduce.

This blog post from Aura Walmer regarding a solar eclipse is a good introduction to ways of thinking when designing sonifications. Other examples are here. The techniques demonstrate Open Sonification techniques.

An alternative sonification toolkit by Sarah Lenzi and Paulo Ciuccarelli is available, if you would prefer.

  1. Sonify Data

In this option, you can create a sonification using your own data (or you can use a sample from the Text Creation Partnership's (TCP) EEBO data). There are a list of resources that will, hopefully, appeal to a wide range of interests and skill ranges.

A set of sonifications for the DHOxSS CSV file might be helpful to begin thinking about the data and design possibilities as well as the limitations of some of these tools.

If you are interested in the audio mapping, then you could try the audio mapping tool to create a location based audio map to explore a concept. Here you can use any recording device, like a phone or field recorder, to record a short audio and upload it.

Data

If you do not have any data, you can use sample of the TCP EEBO data.

We will discuss your experiences of working with either task in the final section.
